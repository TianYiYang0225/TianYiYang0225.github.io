2.数据和方法

vm.a == data.a这两者是相同的。

这两者其中之一数据改变，另外一边数据也会改变。

注：data里的数据只有改变才会导致数据重新加载，如果data中加入了一个新的参数，将不会导致数据重新加载，如果需要预留项，可以先定义为空或者其他值，在要’加入’时，修改数据即可。

之前介绍的都是v-if现在来介绍一下v-show

v-show的使用方式跟v-if非常相似

ok是true是显示，是false时不显示。

但是区别在于，不管它是true还是false都会被渲染只不过对于你来说只是看和看不到的问题。

v-if和v-show来做区分的话就是，v-if像是存在与不存在，v-show就是显示和不显示。

搭建个人blog

准备工作

git

https://git-scm.com/downloads 安装过程不过多描述

node

https://nodejs.org/zh-cn/ 安装过程不过多描述

github

https://github.com/

hexo

https://hexo.io/

简单来说就是在node安好的前提下

npm install -g hexo-cli 一行就行了

搭建环境

  1. Github 注意事项

弄好ssh秘钥

  1. 创建个人仓库

仓库命名时一定要注意

用户名+github.io

因为我有了这个仓库所以是重复创建，正常情况下应该是绿色的。

切记

名字一定要这么起，要是起别的，就做好要输很长的一段链接的准备吧

创建好仓库之后在setting中设置github pages

  1. 学习hexo操作

    初始化项目

hexo init 项目名 例：hexo init blog init可以简写为i

​ 切换根目录

cd 项目名

​ 构建代码

hexo generetor 例：hexo g 也可以

​ 部署服务

hexo server 例：hexo s 也可以

成功的话即可通过localhost:4000看到hexo欢迎界面

如果对theme没有要求的话用默认即可，如果想自定义的话，可以去官网找一些好的模板，我用的是next

如果想套模板的话

以next为例 在当前项目根路径下

完成后，打开根目录下的_config.yml文件

把theme: landscape改成 theme: next

然后清除一下缓存

hexo clean

重新生成下代码

hexo g

再开服务看的话就可以看到主题有了变化

hexo s

next有三种主题样式

分别是Muse Mist Pisces三种模式

可以在项目根目录/themes/next/_config.yml里面找到参数scheme进行修改

  1. 同步到github上

为什么提前写这步，因为努力了半天不就是想看效果吗，所以先看个效果给自己打个鸡血。

先修改自己的根目录下的配置文件(_config.yml)

一般在最后找到deploy:

type: git
repository: https://github.com/TianYiYang0225/TianYiYang0225.github.io.git
branch: master

效果直观不？记得冒号后面一定要加一个空格，格式一定不能改变。

然后在根目录安装插件

npm install hexo-deployer-git –save

安装好的话，直接

hexo d 让本地项目部署在github仓库上

然后再看看自己的网站，

你就会发现，看我这说了半天一定是有用的。

  1. 写文章

hexo new “文章名”

默认是markdown写法，有需要的可以提前学习一下。

文件生成在根目录/source/_posts里面

写完之后记得，一定要有这两步，

hexo g

hexo d

很关键， hexo g就相当于重新构建本地代码，hexo d就是同步远程github库。

就是这样，所有操作都很好理解，只要用心就好了，以后每次写文章记得 hexo g和hexo d就行。

关于主题名字啊，作者啊这些，可以在主题的配置文件里，进行专门的修改

有不会的请留言~具体深度定制请关注下一篇文章~~~
